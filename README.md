# Traffic Accident Hotspot Analyzer

A data analytics project that processes 3M+ US road accident records to identify dangerous hotspots, visualize risk patterns, and predict accident risk levels for any location and time.

---

## What this does

Raw accident data — latitude, longitude, severity, time, weather — gets transformed through a pipeline that answers three practical questions:

- **Where** are accidents clustering? (DBSCAN geospatial clustering)
- **When and why** do they happen? (EDA across time, weather, and road features)
- **How risky** is a specific location right now? (Random Forest classifier)

The output is a Django dashboard with an interactive map, insight charts, and a live risk prediction tool.

## Pipeline overview

**Notebook 01 — EDA**
Loads the [US Accidents dataset (Kaggle)](https://www.kaggle.com/datasets/sobhanmoosavi/us-accidents), handles missing values, engineers time features (hour, day, season), groups weather into categories, and produces charts covering accident distribution by time, severity, weather, and geography.

**Notebook 02 — Clustering**
Runs DBSCAN with haversine distance on the top cities by accident count. Each cluster becomes a labelled hotspot with a risk level (High / Medium / Low) derived from accident density and mean severity. Outputs interactive Folium maps — static heatmap, time-animated heatmap by hour, and a layered hotspot map with popups.

**Notebook 03 — Prediction**
Encodes features (cyclical sin/cos for hour and month, one-hot for weather and time-of-day), trains a Random Forest classifier with class-balancing, evaluates with confusion matrix + ROC curves, and saves `rf_model.pkl` + `feature_meta.json` for the Django API.

**Django dashboard**
Three pages served from the saved artifacts — no retraining needed at runtime.

## Key findings

- Accidents peak at **8am and 5–6pm**, consistent with commute patterns
- **Friday** has the highest weekly count; weekends skew toward higher severity
- **Clear weather** accounts for the most accidents by volume, but **Snow/Ice** produces the highest mean severity
- California, Florida, and Texas account for a disproportionate share of total incidents
- Junctions and crossings correlate with elevated severity even controlling for volume

## Running locally

**Requirements:** Python 3.10+, pip

```bash
# 1. Clone the repo
git clone https://github.com/pratee-1110/traffic_nalyser.git
cd traffic_analyser

# 2. Create a virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Download the dataset
# Create a Kaggle account, download the API token from kaggle.com → Account
# Place kaggle.json at ~/.kaggle/kaggle.json, then:
kaggle datasets download -d sobhanmoosavi/us-accidents
unzip us-accidents.zip -d /kaggle/input/datasets/sobhanmoosavi/us-accidents/

# 5. Run the notebooks in order
jupyter lab
# Run: 01_eda.ipynb → 02_clustering.ipynb → 03_prediction.ipynb
# This generates data/ models/ and maps/ directories

# 6. Start the Django server
cd dashboard
python manage.py runserver
```

Open `http://127.0.0.1:8000`

**Note:** The notebooks take ~10–15 minutes to run on the full 3M row dataset. If you want to test the dashboard quickly, run with a sampled subset — set `nrows=100000` in the `pd.read_csv()` call in notebook 01.
